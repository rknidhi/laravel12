Commands
=========
1. docker compose up -d
  Starts containers in detached mode
  Uses existing images (no rebuild)
  Fast startup
  Changes in Dockerfile are ignored

2. docker compose up -d --build
  Forces Docker to rebuild images before starting
  Then runs containers in detached mode
  Slower, but ensures latest changes are included
  👉 Use this when:
    You changed Dockerfile
    You updated dependencies
    You modified build-related config

Pro tip (saves time)
====================
If you're actively developing:
  docker compose up -d --build --remove-orphans
  --remove-orphans cleans up unused containers from old configs
